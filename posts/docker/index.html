Docker

Introduction to Docker
July 11, 2023 · Gabriel Glösmann

General#

Docker is an open-source platform that enables software to be packaged into containers. Containers provide a consistent and portable environment to run applications and their dependencies across different computing environments. Docker offers a lightweight, isolated, and reproducible way of deploying and managing applications, making it easier to develop, deliver, and scale applications in various environments, such as local machines, cloud servers, or clusters.

Docker has three different types:

  1. Dockerfile
  2. Docker Image
  3. Docker Container

Dockerfile#

A Dockerfile is a text file that contains a set of instructions used to build a Docker image. It defines the base image, sets up the necessary dependencies, copies files into the image, and specifies the commands to be executed when the container is run. By using a Dockerfile, you can automate the creation of Docker images and ensure consistency across different environments.

Here are some commonly used instructions in a Dockerfile:

  • FROM: Specifies the base image for your Docker image.
  • ﻿MAINTAINER (optional): Sets the author/maintainer of the Dockerfile.
  • RUN: Executes a command during the build process.
  • ﻿COPY or ﻿ADD: Copies files or directories from the host machine to the Docker image.
  • ﻿WORKDIR: Sets the working directory inside the Docker image.
  • ﻿ENV: Sets environment variables inside the Docker image.
  • ﻿EXPOSE: Exposes a specific port for communication with other containers or the host machine.
  • CMD or ﻿ENTRYPOINT: Specifies the command to run when a container is started based on the image.

Example:

FROM alpine:3.13
RUN apk upgradel
RUN apk add nginx
COPY files/default.conf /etc/nginx/conf.d/default.conf
RUN mkdir -p /var/www/html
WORKDIR /var/www/html
COPY --chown=ngninx:nginx /files/html .
EXPOSE 80
CMD ["nginx", "-g", "pid /tmp/nginx.pid; daemon off;"]

Docker Image#

A Docker image is a lightweight, standalone, and executable package that contains everything needed to run a piece of software, including the code, runtime environment, system tools, libraries, and dependencies. It is built from a Dockerfile, which is a text file that provides instructions on how to assemble the image. Docker images are stored in a registry and can be shared, distributed, and deployed on Docker containers across different environments.

Use the following command to build the docker image:

docker build . -t web

Check the status of the docker image:

docker image ls

Docker Container#

A Docker container is a lightweight, portable, and self-sufficient software package that contains everything needed to run an application, including the code, runtime environment, system tools, and libraries. It uses containerization technology to isolate the application and its dependencies from the underlying infrastructure, allowing the application to run consistently across different computing environments. Docker containers provide a consistent and reproducible method for packaging, distributing, and deploying software applications.

Use the following command to run a docker container:

docker run -dt -p 80:80 --name web01 web

The ﻿docker run command is used to create and start a new container from a specified Docker image. Here is an overview of some important options and parameters commonly used with ﻿docker run:

  • Image: The Docker image name or ID from which to create the container.
  • Detached mode: ﻿-d option allows the container to run in the background.
  • Container name: ﻿–name option assigns a name to the container.
  • Port mapping: ﻿-p option maps container ports to the host machine in the format ﻿host_port:container_port.
  • Environment variables: ﻿-e option sets environment variables inside the container.
  • Volume mapping: ﻿-v option maps host directories or volumes to container directories.
  • Resource constraints: ﻿–cpus and ﻿–memory options limit CPU and memory usage by the container.
  • Container removal: ﻿–rm option automatically removes the container when it finishes running.
  • Container interaction: ﻿-it option provides an interactive terminal for the container.

Run a bash inside a Docker container:

docker exec -ti CONTAINER_ID bash

Docker Hub#

Docker Hub is a cloud-based repository where individuals and organizations can store, distribute, and manage containerized applications. It serves as a centralized platform for sharing and discovering Docker images. Docker images, which are lightweight, standalone executable packages containing everything needed to run an application, can be pulled from Docker Hub to deploy and run containers on various platforms.

Push a docker image to a private repro>

docker tag my-image username/my-repo
docker push username/my-repo

Use Docker pull to download an image or an repository:

docker pull [OPTIONS] NAME[:TAG|@DIGEST]
